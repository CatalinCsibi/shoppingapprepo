How to test spring boot application

As the app is configured to use Keycloak we will need to use Bearer tokens generated by it in order to be able to perform any operation

1. Run the application


2. After configuring the Keycloak client as shown in the README file for Keycloak we will need to ask for an authorization code:
   We will need to open a browser window and use a url as the following:
   http://localhost:8080/realms/shopping-realm/protocol/openid-connect/auth?client_id=shopping-app-client&response_type=code&scope=openid profile&redirect_uri=http://localhost:8083/callback (you will just need to paste this url in the browser)

   Here the client_id and redirect_uri should match the values that you have configured when creating the client in Keycloak.

    After pasting the url you will be redirected to the redirect uri and you will se something like this:
   http://localhost:8083/callback?session_state=NN9RC0N2BxxWe0VX8pFnmzZO&iss=http%3A%2F%2Flocalhost%3A8080%2Frealms%2Fshopping-realm&code=a2840ba1-946b-0dbc-00b9-b4bde3f64fdf.NN9RC0N2BxxWe0VX8pFnmzZO.1a129d3d-251e-4ecd-abac-2f147f936064

    At the end we can see the authorization code: code=a2840ba1-946b-0dbc-00b9-b4bde3f64fdf.NN9RC0N2BxxWe0VX8pFnmzZO.1a129d3d-251e-4ecd-abac-2f147f936064


3. Once we have the authorization code we will need to exchange it for a jwt token. We will need to make a post request to the following URL (use Postman for this):
   http://localhost:8080/realms/shopping-realm/protocol/openid-connect/token
    
    The realm here should match the one that you have created during the keycloak configuration. So change shopping-real with the name that you have used.
    If you've used the same name keep the url as is.
    
    When making the request we need to also add a request body under the form of x-www-form-urlencoded.

    Here we need to add the following parameters: 

   grant_type:authorization_code

   client_id:shopping-app-client

   client_secret:oVQs7ZFSKvS32Ty1k1gMHQvWWZ6bKzSS

   code:a2840ba1-946b-0dbc-00b9-b4bde3f64fdf.NN9RC0N2BxxWe0VX8pFnmzZO.1a129d3d-251e-4ecd-abac-2f147f936064

   redirect_uri:http://localhost:8083/callback

   scope:openid profile

    The client secret can be retrieved from Keycloak:
    
    a. Login to Keycloak with you admin user (admin / admin) 

    b. Navigate to your realm (we shouldn't be on master)

    c. Go to clients and select your client that you have previously configured (shopping-app-client for example)

    d. Go to credentials, copy the client secret and paste it in the x-www-form-urlencoded form

    The code will be retrieved from the redirect URI as seen above: code=a2840ba1-946b-0dbc-00b9-b4bde3f64fdf.NN9RC0N2BxxWe0VX8pFnmzZO.1a129d3d-251e-4ecd-abac-2f147f936064 (just copy paste what comes after equal sign and paste it in the form)

5. Once we have the access token we can make requests to our API using Postman:

    
    First of all, all requests should be authorized. So we need to go to authorization tab for each request, select Bearer Token and paste the access token there

    Example POST request:

    URL: http://localhost:8090/api/v1/products

    BODY (JSON):
    
        {"name": "iPhone 17", "description": "Latest Apple phone", "price": 999.99, "stockQuantity": 25, "category": "phones", "imageUrl": "https://example.com/iphone.jpg"}

    Eample GET all request:
    
    URL: http://localhost:8090/api/v1/products?page=0&size=10

    Example GET by name request: 

    http://localhost:8090/api/v1/products?name=iphone&page=0&size=5
